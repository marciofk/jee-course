Lab 1: HTTP Protocol

This lab contains a series of exercises related to the HTTP protocol.

Prerequisites:
  • HTTP Client: We recommend Postman. If you do not have Postman yet, please download and install it using this link
  • Having the mock REST API running. To know how to run it, please refer to the Instructions (Starting the mock REST API)
Exercise 1

Execute a GET request to the following address: http://localhost:8090/

Questions
  1. 1.1. What is the MIME type of the response?
  2. 1.2. Which header specifies the MIME type of the response?
  3. 1.3. Which header specifies the response size?
  4. 1.4. What is the unit to describe the response size?
Exercise 2

Execute a GET request to the following address: http://localhost:8095/api/customers

Questions
  1. 2.1 What is the MIME type of the response?
Exercise 3

Execute a GET request to the address http://localhost:8080/JEECourse/module02/example02, with the following parameters:

  • name: John Doe
  • company: C&A

The mime type must be application/x-www-form-urlencoded

Questions
  1. 3.1 What is the MIME type of the response?
  2. 3.2 What is the UTF-8 character code of the character &
  3. 3.3 What is the content of the HTTP response body?
  4. 3.4 What is the status code?

Questions
  1. 4.1 What is the final URL?
  2. 4.2 What is the value of the parameter company?
Exercise 5

Execute a PUT request to the following address: http://localhost:8095/api/customers

The MIME type of the request must be application/json, and the body must be:

{
   "firstName": "john",
   "lastName": "doe",
   "gender": "male",
   "address": "1234 Anywhere St.",
   "city": " Phoenix ",
   "state": {
      "abbreviation": "AZ",
      "name": "Arizona"
   },
   "latitude": 33.299,
   "longitude": -111.963
}
Questions
  1. 5.1 What is the content of the response body
  2. 5.2 Is there a difference between the request and the response body?
  3. 5.3 What happens if you do not send the correct MIME type?
