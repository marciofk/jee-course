Lab 5: Cookies

In this exercise, you will build a servlet that deals with cookies (sending and retrieving)

Prerequisites:
  • Configuring the server-side project: If you do not have it, follow the installation instructions
  • Starting the client-side project: If you do not have it, follow the instructions
Part 1: Understanding the exercise

Open the client application and login with a valid user and password.

For this exercise, you will use cookies to store the last login for further usage, as well as track the number of successful login attempts.

Part 2: Changing the Login servlet

In your server-side project, modify the Login servlet to:

  1. If the user and password are valid, create and send a cookie called email, containing the parameter sent by the user. This cookie must be valid for one day and all paths of the host (see the method setPath of Cookie)
  2. To track successful attempts, use a cookie called visits. First, check if the client sent this cookie. If yes, retrieve the value, increment it, and send back the cookie to the client. If the cookie is not present in the request, it means that this is the first time that the client is logging in. If this is the case, assume the value as zero

Part 3: Testing the servlet

Access the endpoint using Postman. Check if the servlet is returning the cookies as headers

Part 4: Testing the servlet from the client app

Test the authentication process to check if the last successful login is present in the form, as well as the number of successful attempts.
